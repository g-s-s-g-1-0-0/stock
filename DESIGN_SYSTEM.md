# 공수성가 디자인 시스템

이 문서는 새 UI를 만들거나 기존 UI를 고칠 때 우선 참고하는 공통 기준입니다. 새 패턴을 만들기보다 여기의 토큰, 타이포, 폼, 검증, 테이블 규칙을 먼저 재사용합니다.

## 원칙

- 정보는 `핵심 수치 -> 보조 설명 -> 행동` 순서로 배치한다.
- 금융 수치는 오른쪽 정렬, 본문과 상태 설명은 왼쪽 정렬을 기본으로 한다.
- 클릭 가능한 텍스트는 필요한 곳에만 쓰고, 반복 설정처럼 주요 행동은 모달로 모은다.
- 에러는 입력 즉시 보여주고, 저장 버튼을 누른 뒤에야 처음 보이게 하지 않는다.

## 타이포

- 섹션 제목: 굵게, 짧게, 한 줄 기준.
- 라벨: 12px, muted 컬러, 800 weight.
- 본문 설명: 기본 폰트, muted 컬러, 1~2줄 이내.
- 검증 메시지: 12px, 400 weight. 에러는 red, 안내는 muted.
- 숫자/금액: tabular number를 우선하고, 테이블 안에서는 오른쪽 정렬한다.

## 폼

- 입력 필드는 `.login-field` 패턴을 우선 재사용한다.
- `input`, `select`는 동일한 높이, border, radius, focus ring을 유지한다.
- 모달 설명은 1줄을 우선하고 길면 2줄까지만 허용한다.
- 검증 메시지는 입력 박스 아래 `6px` 이상 간격을 두고 배치한다.
- 검증 메시지는 굵게 처리하지 않는다.

## 검증

- 사용자가 입력하는 즉시 검증 메시지를 갱신한다.
- 저장 버튼은 명백히 저장 불가능한 값일 때 비활성화한다.
- 반복 입금일처럼 일부 기간에만 적용되지 않는 값은 에러가 아니라 안내로 표시한다.
- 월간 입금일은 `1~31` 정수만 허용한다. `29일`, `30일`, `31일`은 해당 날짜가 없는 달에는 입금되지 않는다는 안내를 표시한다.

## 테이블

- 금액, 수익률, 가격은 `.number-cell`을 사용한다.
- 상태 배지는 컬러 의미를 일관되게 유지한다. 실패 계열은 같은 실패 컬러를 쓴다.
- 컬럼을 고정할 때는 고정 전 실제 너비를 유지한다.
- 테이블에 새 컬럼을 추가하면 빈 행의 colspan/셀 개수도 함께 맞춘다.

## 모달

- 제목은 사용자가 하려는 행동을 그대로 쓴다.
- 닫기, 취소, 저장 버튼 위치와 스타일은 기존 confirm modal 패턴을 따른다.
- 금액/주기처럼 같은 목적의 입력은 하나의 모달에서 처리한다.
- 과도한 설명 대신 예측 가능한 기본값과 즉시 검증을 제공한다.

## 트레이딩 로그

- 요약 수치 순서는 `총 투자 기간 -> 보유 현금 -> 총 투자금 -> 예상 손익`을 기본으로 한다.
- 투자금은 직접 입력하지 않고, 설정된 현금 투입 규칙과 매수 시점 보유 현금의 50% 규칙으로 자동 계산한다.
- 보유 현금 클릭은 현금 직접 수정이 아니라 `초기 자금`, `입금 주기`, `입금일/요일`, `입금액` 설정으로 연결한다.
